# Zac & Casey — Portfolio Prototype

A static portfolio prototype. Pure HTML/CSS/JS — no build step.

## Working with the AI (Zac's note to self)

Treat GitHub like Dropbox for this project. Just talk to Claude / Codex in plain English — no git commands needed:

- **"Save my changes online"** — pushes to GitHub.
- **"Pull the latest changes"** — gets the newest version.
- **On a new machine:** _"Set up my portfolio project from https://github.com/zacnairn3-dot/zac-and-casey-portfolio"_

The AI handles all the git stuff.

## Run locally

From this folder:

```sh
python3 -m http.server 8766
```

Then open <http://localhost:8766>.

## Structure

- `index.html` — the page shell (rail nav, hero, case studies container, awards, contact).
- `script.js` — renders case studies from the `projects` array, wires the rail's scroll-active highlight, and handles hash navigation.
- `styles.css` — all styling.
- `assets/` — images, video, and the tape/paper textures generated by `_tools/`.
- `_tools/` — Python scripts that generate the photographic tape and paper textures (Pillow). Optional; the design currently doesn't use those generated images, but the scripts are kept for reference.

## Editing the project list

All eight case studies live in `script.js` in the `projects` array. Each entry has an id, client, title, line, role, result, color, image/thumb paths, and a `media` array with the campaign assets in display order.

## Styling notes

- `--red` is the global accent and gets retinted on scroll to whichever case is in view (see `setActiveProject` in `script.js`).
- Each case has a `--tilt` custom property for its slight tilt and an inline `--red` for the project color.
- Body background is a paper cream + faint vertical-rule pattern + soft yellow corner glow + a subtle dot-grain `.noise` overlay — keep these as a stack to preserve the textured-paper feel.
